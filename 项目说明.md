## 📌 项目简介

这是一个**从零开始**的语音控制系统，覆盖了：

- **硬件**：原理图设计、PCB  Layout、焊接调试
- **固件**：STM32 裸机开发、ADC+DMA 采集、串口通信
- **算法**：MFCC 特征提取、CNN 模型训练、实时推理
- **PC 端**：Python 服务器、数据采集、模型部署

**核心链路**：
```
声音 → MAX9814 麦克风 → STM32F103 采集 → HC-06 蓝牙 → PC 端 MFCC+CNN → 识别结果
```

## 🛠️ 硬件设计

### 核心硬件

| 模块   | 型号                 | 说明                |
| ------ | -------------------- | ------------------- |
| 主控   | STM32F103C8T6 核心板 | 原型验证用          |
| 麦克风 | MAX9814 模块         | 自带 AGC，模拟输出  |
| 蓝牙   | HC-06                | 9600bps，最稳定     |
| 屏幕   | 0.96寸 OLED          | I2C 接口，0x78 地址 |
| 电源   | 5V 降压模块          | 电池供电方案        |

### 引脚分配

| 模块   | 信号 | STM32 引脚       | 备注                |
| ------ | ---- | ---------------- | ------------------- |
| 麦克风 | OUT  | PA0              | ADC 输入            |
|        | VDD  | 3.3V             | 加去耦电容          |
|        | GND  | GND              | 模拟地单独走        |
| 蓝牙   | TXD  | PB11 (USART3_RX) | 模块发送→STM32 接收 |
|        | RXD  | PB10 (USART3_TX) | STM32 发送→模块接收 |
|        | VCC  | 3.3V             | 加去耦电容          |
| OLED   | SCL  | PB6              | I2C 时钟            |
|        | SDA  | PB7              | I2C 数据            |
|        | VCC  | 3.3V             | 加去耦电容          |


### PCB 设计要点

- **电源线加粗**：5V/3.3V 走 27mil
- **去耦电容**：每个模块 VCC 旁放 10μF+0.1μF，紧贴引脚
- **模拟地处理**：MAX9814 GND 单独走，单点接数字地

### MFCC 特征提取

MFCC 是语音识别的黄金标准，主要优势：

- **模拟人耳**：梅尔刻度让机器“听得像人一样”
- **大幅降维**：16000 点 → 1300 维，压缩 12 倍
- **抗噪性强**：整个流程天然滤掉噪声




### 模型训练

- **模型**：3 层 Conv1D + 全连接
- **数据**：Google Speech Commands + 自采数据
- **准确率**：93%（测试集）
- **输出**：`speech_cnn.h5`

## 🧠 经验总结

### 硬件

1. **以实物为准**：MAX9814 引脚排序
2. **电源线要粗**：20-30mil 起步
3. **电容要贴芯片放**：0.1μF 必须紧贴引脚

### 软件

1. **裸机够用就别急着上系统**
2. **帧头同步很重要**：`AA 55` 解决粘包问题
3. **数据分包比降低速率更有效**

### 算法

1. **MFCC 参数不能错**：训练和推理必须一致
2. **CMSIS-DSP 有坑**：arm_bitreversal 自己手写 fix
3. **数据增强有用**：加了噪声后识别率提升明显
