> 📅 **版本**：F103原型验证版  
> 🔧 **主控**：STM32F103C8T6核心板   
> ⚠️ **注意**：蓝牙用的是 **USART3**，且 **TXD/RXD 是交叉连接**

### 🔌 蓝牙接线特别说明

**这是最容易接错的地方，一定要看仔细：**

| HC-06 引脚      | 接 STM32 引脚        | 说明                  |
| --------------- | -------------------- | --------------------- |
| TXD（模块发送） | **PB11 (USART3_RX)** | 模块发送 → STM32 接收 |
| RXD（模块接收） | **PB10 (USART3_TX)** | STM32 发送 → 模块接收 |

**口诀**：**TXD 接 RX，RXD 接 TX**（交叉连接）

| 信号       | 引脚 | 注意事项                 |
| ---------- | ---- | ------------------------ |

#### 数字通信
| 接口      | 引脚 | 功能                   | 速率    |
| --------- | ---- | ---------------------- | ------- |
| USART3_TX | PB10 | 蓝牙发送（STM32→模块） | 9600bps |
| USART3_RX | PB11 | 蓝牙接收（模块→STM32） | 9600bps |
| I2C1_SCL  | PB6  | OLED时钟               | 100kHz  |
| I2C1_SDA  | PB7  | OLED数据               | 100kHz  |

### 🎯 选型理由

#### PB10/PB11 做蓝牙
- 用 USART3 避开 USART1（PA9/PA10 留着备用）
- 核心板引出这些脚，不用飞线
- HC-06 默认 9600bps，正好匹配

#### 为什么 TXD 接 PB11？
- HC-06 的 TXD 是**发送**数据
- STM32 的 PB11 是 USART3 的 **RX（接收）**
- 一个发一个收，正好对上

---

### 📐 PCB 布局建议

1. **PA0 麦克风输入**
   - 尽量靠近麦克风模块
   - 周围不要走数字信号线
   - 最好用地线包起来

2. **PB10/PB11 蓝牙**
   - 放在板子边缘（天线朝外）
   - 天线下方不要铺铜、不要走线
   - 远离电机、电源等干扰源

3. **PB6/PB7 OLED**
   - 放在方便插屏幕的位置
   - I2C 走线可以长一点，没问题

4. **电源线**
   - 5V_IN 和 3.3V 走 20-30mil
   - 每个模块旁边放 10μF+0.1μF 去耦电容

### 📝 代码里对应的配置

```c
// ADC 初始化（PA0）
ADC_ChannelConf.Channel = ADC_CHANNEL_0;

// USART3 初始化（PB10 TX, PB11 RX）
USART3_BaudRate = 9600;  // 和 HC-06 匹配
GPIO_Pin_10 = PB10;      // TX（STM32发送）
GPIO_Pin_11 = PB11;      // RX（STM32接收）

// 发送一个字节
void HC06_SendByte(uint8_t data)
{
    USART_SendData(USART3, data);
    while(USART_GetFlagStatus(USART3, USART_FLAG_TXE) == RESET);
}

// 接收一个字节
uint8_t HC06_ReceiveByte(void)
{
    while(USART_GetFlagStatus(USART3, USART_FLAG_RXNE) == RESET);
    return USART_ReceiveData(USART3);
}
```
