## 为什么用 MFCC？——语音识别特征提取的选择过程

> 📅 **时间**：项目初期  
> 🔧 **涉及**：特征提取、模型训练  
> 🎯 **目的**：解释为什么最终选了 MFCC，而不是别的特征

---

### 🤔 问题的起点

做语音识别的第一个问题就是：**怎么把声音送给神经网络？**

你不能直接把原始音频扔给模型，因为：
- 1秒钟的音频，16000Hz采样率，就是 **16000个数字**
- 这16000个点直接作为输入，模型参数会爆炸
- 更重要的是：**原始波形里有很多信息对语音识别是没用的**

所以必须做 **特征提取**——把16000个点，压缩成一小撮真正有用的特征。

---



### ✅ 为什么 MFCC 胜出了

#### 理由1：模拟人耳听觉特性

人耳对频率的感知不是线性的：
- 低频（<1000Hz）：分辨力高，能听出细微差别
- 高频（>1000Hz）：分辨力低，差别大也听不出来

**MFCC 做了什么事**：
- 用梅尔刻度（Mel scale）重新标定频率轴
- 低频区域多放些滤波器，高频区域少放
- **让机器“听得像人一样”**

这个特性让我在测试时发现：同样是在嘈杂环境，MFCC 的识别率明显比 FFT 高。

#### 理由2：大幅降维

以 1 秒音频为例：

| 特征类型 | 维度             | 数据量        |
| -------- | ---------------- | ------------- |
| 原始波形 | 16000 点         | 32000 字节    |
| FFT 频谱 | 256 × 100帧      | 25600 字节    |
| **MFCC** | **13维 × 100帧** | **2600 字节** |

MFCC 把数据量压缩了 **12倍**，而且保留的是语音识别最需要的信息。

这对我的项目特别重要，因为要用 9600bps 的蓝牙传数据，数据量越小越好。

#### 理由3：抗噪能力强

MFCC 的提取流程里有几个天然抗噪的步骤：

| 步骤                | 作用                       |
| ------------------- | -------------------------- |
| 预加重              | 提升高频，补偿口唇辐射     |
| 分帧加窗            | 平滑信号，减少突变         |
| 梅尔滤波            | 模拟人耳，对噪声不敏感     |
| 取对数              | 压缩动态范围，弱化噪声影响 |
| DCT（离散余弦变换） | 去相关，保留主要特征       |

我做过对比测试：同样在风扇噪音环境下，FFT 的识别率掉到 60%，MFCC 还能维持在 85% 以上。

#### 理由4：业界标准，遇到问题有答案

- TensorFlow 官方语音教程：用 MFCC
- Edge Impulse：MFCC 是默认选项
- 几乎所有语音识别论文：都用 MFCC
- librosa、python_speech_features 等库：都支持 MFCC

这意味着：用 MFCC 遇到问题，一搜就有答案。这对独立做项目的我来说太重要了。

---

### 📊 实测对比（在我自己的数据集上）

| 特征     | 模型大小  | 训练时间   | 安静环境准确率 | 嘈杂环境准确率 |
| -------- | --------- | ---------- | -------------- | -------------- |
| 原始波形 | 10MB+     | >10小时    | 85%            | 40%            |
| FFT 频谱 | 3MB       | 2小时      | 88%            | 55%            |
| **MFCC** | **800KB** | **30分钟** | **95%**        | **85%**        |

**结论**：MFCC 在模型大小、训练时间、准确率、抗噪能力上，全面胜出。

---

### 📝 MFCC 提取流程（我代码里怎么做的）

```
原始音频 (16000Hz, 1秒)
    ↓ 预加重 (y[n] = x[n] - 0.97*x[n-1])
    ↓ 分帧 (每帧25ms，步长10ms → 101帧)
    ↓ 加窗 (汉明窗)
    ↓ FFT (512点)
    ↓ 梅尔滤波 (40个滤波器)
    ↓ 取对数 (log)
    ↓ DCT (保留前13个系数)
    ↓ MFCC特征 (13维 × 101帧)
    ↓ 转置 (101 × 13) ← 喂给CNN的输入形状
```

**关键参数**（试出来最稳的）：
```python
librosa.feature.mfcc(
    y=audio,              # 原始音频
    sr=16000,             # 采样率
    n_mfcc=13,            # MFCC系数个数（13是标准值）
    n_fft=512,            # FFT窗口大小
    hop_length=160,       # 帧移（10ms = 160点）
    win_length=400,       # 帧长（25ms = 400点）
    window='hamming',     # 窗函数
    n_mels=40             # 梅尔滤波器个数
)
```

### 🕳️ 踩过的坑

#### 坑1：参数不一致

**现象**：训练时识别率 95%，部署到实时识别只有 30%

**原因**：训练用的 MFCC 参数和推理时的不一样

```python
# 训练时（正确）
mfcc = librosa.feature.mfcc(y=audio, sr=16000, n_mfcc=13, hop_length=160)

# 推理时（错误，用了默认参数）
mfcc = librosa.feature.mfcc(y=audio, sr=16000)  # 默认 hop_length=512
```

**解决**：把 MFCC 参数写成配置文件，两端用同一份。

#### 坑2：维度顺序

**现象**：模型报错说输入形状不对

**原因**：librosa 默认输出 (n_mfcc, 帧数)，CNN 期望 (帧数, n_mfcc)

**解决**：
```python
mfcc = mfcc.T  # 转置一下
```

#### 坑3：arm_bitreversal 链接不上

**现象**：编译时报错找不到 arm_bitreversal_32

**原因**：CMSIS-DSP 库的版本问题，汇编文件没链接对

**解决**：自己手写了 `arm_bitreversal_fix.c`，用 C 实现这两个函数。

这个坑我专门写了个文档：`arm_bitreversal由来`

#### 坑4：MFCC 个数选错

**现象**：模型训练 loss 降不下去，准确率卡在 60%

**原因**：n_mfcc 设得太小（比如 8），丢失信息

**解决**：试了 8、13、20、30，**13 效果最好**，20 以上准确率反而下降（引入了噪声）

---

### 🎯 结论

**用 MFCC 不是因为“流行”，而是因为：**

1. **它模拟人耳**——梅尔刻度让机器“听得像人一样”
2. **它大幅降维**——16000点 → 1300维，压缩12倍
3. **它抗噪声**——整个提取流程天然滤掉噪声
4. **它是业界标准**——有成熟的库、参数、论文可参考
5. **它被验证过**——几十年的语音识别研究，MFCC 是沉淀下来的精华

对我来说还有一个特别重要的点：**MFCC 的数据量小**，让我能在 9600bps 的蓝牙带宽下传完整数据。
