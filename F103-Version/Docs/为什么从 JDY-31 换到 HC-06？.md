## 为什么从 JDY-31 换到 HC-06？

> 🔧 **涉及模块**：蓝牙通信模块

---

### 🤔 为什么会有这次更换？

最开始用的是 **JDY-31**，选它的理由是：

- 支持蓝牙4.0
- 标称距离 30 米
- 价格便宜

但实际用下来，问题一堆...

---

### 🐛 JDY-31 遇到的真实问题

#### 问题1：115200bps 根本稳不住

之前为了传 MFCC 特征，我必须要用 115200bps。但 JDY-31 在这个速率下：

| 现象                 | 发生频率   |
| -------------------- | ---------- |
| 传输几十秒后卡死     | 每30秒一次 |
| 数据错位，帧头对不上 | 经常       |
| 需要不断重启模块     | 烦死       |
| PC端收到乱码         | 偶尔       |

查资料发现，JDY-31 虽然标称支持 115200，但**实际稳定工作的点就是 9600bps**。很多帖子里都提到：“JDY-31 用 9600 最稳，115200 容易丢包”。

#### 问题2：蓝牙4.0 的兼容性问题

JDY-31 是基于 **CC2541 芯片**的蓝牙4.0模块。理论上应该更好，但实际：

- 和我的老手机配对偶尔失败
- 在 Windows 上配对后虚拟COM口有时不出来
- 需要手动配置传出端口，步骤繁琐

#### 问题3：6个引脚，焊接麻烦

JDY-31 有 6 个引脚（STATE、RXD、TXD、GND、VCC、EN）。虽然 EN 可以上拉，但在 PCB 上多一个引脚就是多一份麻烦。

---

### ✅ 为什么换 HC-06？

#### 理由1：9600bps 下极其稳定

HC-06 是基于 **BC417 芯片**的蓝牙2.0模块。虽然是老技术，但在 9600bps 这个速率下：

- **连续传 24 小时不卡死**
- 几乎没有丢包
- 数据从来不错位

HC-06 蓝牙模块具有较低功耗、低成本的高性能无线收发系统。HC-06 在配对时电流为 30–40 mA，通信时降至 8–10 mA，适合电池供电场景。

#### 理由2：4个引脚，焊接简单

HC-06 只有 4 个引脚（VCC、GND、TXD、RXD）。没有 EN、没有 STATE，PCB 上直接拉线就行，少两个引脚少两个出错的点。

#### 理由3：即插即用，无需折腾

- 默认波特率就是 **9600**
- 默认配对密码 `1234`
- 在 Windows 上配对后自动出 COM 口，不用手动配置
- 和 STM32 的连接和普通串口一模一样

#### 理由4：便宜且够用

HC-06 基于蓝牙2.0协议，采用透传模式，通过UART接口与微控制器通信，实现设备间的无线数据交换。对于我的项目来说：

- 距离要求：<10 米（足够了）
- 速率要求：9600bps（够了）
- 功能要求：透传（够了）

花更少的钱，买更稳的体验。

---

### 📊 对比总结

| 对比项       | JDY-31            | HC-06               |
| ------------ | ----------------- | ------------------- |
| 芯片         | CC2541（蓝牙4.0） | BC417（蓝牙2.0）    |
| 引脚数       | 6个               | **4个**             |
| 稳定波特率   | 9600bps           | **9600bps（极稳）** |
| 配对         | 有时折腾          | **即插即用**        |
| 虚拟COM口    | 需手动配置        | **自动出现**        |
| 价格         | 差不多            | 差不多              |
| **最终结论** | ❌ 不稳定          | ✅ 稳如老狗          |

---

### 🧠 核心经验总结

1. **新不一定比旧好**  
   蓝牙4.0（JDY-31）听起来比蓝牙2.0（HC-06）先进，但在 9600bps 这个低速场景下，老的 HC-06 反而更稳。

2. **引脚越少，麻烦越少**  
   4 个引脚的 HC-06，PCB 上直接拉线就行。少两个引脚，少两个出错的点。

3. **9600bps 是 HC-06 的舒适区**  
   HC-06 默认就是 9600，在这个速率下它可以连续跑几天几夜不卡死。

4. **不要迷信标称值**  
   JDY-31 标称支持 115200、距离 30 米，但实际用起来全是坑。HC-06 标称只有 10 米、只有 9600，但真的能做到。
